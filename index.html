DeadJournals has awaken! IT'S ALIVE!!!!

DeadJournals is a tool for making static blogs and webcomic sites using HTML, CSS, and vanilla Javascript. HTML and Javascript (array and dictionary) knowledge is recommended to begin working with DeadJournals.

If you need a little help getting started, there's a tutorial waiting for you!
And don't forget to check out the live demo examples of the different prebuilt themes!
You can also check out the frequently asked questions.

You can check out this website's source code to see how I set it up since it was made using DeadJournals.
You can also check out the DeadJournals repository where you can download the different releases or even make pull request in the development branch.

What's the purpose?

Who needs a site creator with zero coding when you can get a site creator with minimum coding?

I suppose you could say DeadJournals is a waste of time with all the other static site generators and website builders out there, and you may be right. But isn't the idea of owning your site and being able to transport it anywhere without having to pay to download it or get the watermark removed pretty neat? Plus, DeadJournals is merely a simplistic shell; if one has the skill and knowledge, then they can add any feature to their deadjournal they want. The sky is the limit! And for those who don't know much or don't need much, they're able to use DeadJournals as it comes with some possible tweaks. Double plus, you don't have to install different coding languages or libraries to use DeadJournals, unlike static site generators like Jekyll, Hugo, and Django. All you gotta do is download one of the template zips and you're on your way.

I suppose the purpose of DeadJournals is to allow one to make a simple, transportable, blog/site that doesn't need any extra technologies downloaded or installed and has the ability to let one easily add on their own extra features and styling. It is meant to allow hosting the site to be easy and even free on some platforms.
