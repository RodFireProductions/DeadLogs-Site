Download DeadJournals

First thing first is downloading the deadjournals_(theme).zip file. This can be downloaded on the itch.io page or right here. The current theme options: . Note: If you plan on doing all of your own css, it's recommended you start with an Absent theme.

There are two different templates as of right now, the blog template and the comic template. The comic ones will have comic in its file name, while the others will be just the default blog ones.

Once the zip is downloaded, you need to unzip it to a desired location.

Set Up Basic Configurations

Once you've got everything unzipped, you need to begin setting up your script.js file. As you can probably see, that file is riddled with comments. Comments are everywhere. Those comments will explain what each variable does.

I'd suggest filling out the first 36 lines of that file to go ahead and get some of the basic page generation features of DeadJournals customized to you.

Here's what those lines look like without all of those comments. I'll go through each of the sections I labelled here. 

// Section A
let siteName = "My Site!";
let lang = "en";
let siteUrl = "";

// Section B
let socials = [
    ['My Blog', siteUrl]
]

let author = {
    name: 'Your Name',
    title: 'A human on the internet',
    picture: null,
    extra: ['human'],
    line: null
}

Section A

The first variable you get the configure is siteName which is merely the name of your site, like this site is named DeadJournals and Facebook is named Facebook.

Secondly, lang is the language that your site will be in. By default, 'en' (English) is used as the value. You can change this value though if you plan on writing your site in a different language. Here's a source with a reference list for other languages. This language tag helps screen reader technology, search engines, and page formatting.

The last one here is siteUrl. This is the least important of the three. This variable is not used anywhere in the code but the socials array by default. It is meant to be a way to quickly link the home page of your site where you want in the code. You can leave this blank if you don't wish to use it.

Section B

There's two parts of this section, socials and author. These both deal with the auto generator for the about page. When both are completely filled out, it'll generate something that looks a little like this (results vary depending on theme) and it's code: 

let socials = [
    ['My Blog', siteUrl], // In this case, siteUrl is set to this example site's home page
    ['DeadJournals', 'https://deadjournals.deadinsideartist.art'],
    ['GitHub', 'https://github.com/RodFireProductions/DeadJournals/tree/main/examples/absent']
]

let author = {
    name: "White Void",
    title: 'The Purest Abyss',
    picture: "./assets/images/author.png",
    pronouns: 'ze/zir',
    extra: ['demiace', 'swirling'],
    line: '"BE NOT AFRAID" - Biblically Accurate Angels'
}

Page Creation and Navigation

DeadJournals comes with two different archive and navigation systems. The blog system is the one used for a basic blog, while comic system is meant to be used for webcomics. Of course, you could use the blog system for a webcomic, but I made the webcomic one anyways.

The simplist of the two is the blog system, while the comic one needs a bit more effort.

Blog

Make sure you keep all your posts are located in the post folder; this is important. What they're named doesn't matter.

Page Creation

First, you will need to make a copy of the tutorial.html file or just copy and past the contents into a new file. Name the file whatever, in this tutorial it'll be referred to as extra.html, just make sure you put the .html extension at the end.

In your file, look for the section of code that looks like the following. That's where you'll insert your page/post content. You don't need to edit the file in any other place unless you know what you're doing. 

<div class="postcontent">

</div>

Setting Up Navigation/Archive

Now we'll be working in the script.js file again. Please locate this section of the code: 

let postArchive = [
    //['Title', 'Date', 'Location']
]

There's a comment there that already shows you the basic format. The title is simply the title of your post. The date is the day you posted it. I'd just keeping all your date formats the same for all your posts. And lastly the location is where your post file is located relative to the script.js file.

Here's what it would look like if I added the extra.html file I created in the previous section: 

let postArchive = [
    //['Title', 'Date', 'Location']
    ['First post!', '17 Nov 2021', 'post/extra.html']
]

Do know that you need to include a comma at the end of each nested array before making a new one. You'll get a syntax error if you don't. For example: 

    ['First post!', '17 Nov 2021', 'post/extra.html'], // <---
    ['Second post!', '18 Nov 2021', 'post/extra2.html']

Everytime you want to add a new post, make sure you add it at the bottom of the array/list.

Setting Up Navigation/Archive

First things first, we'll be working in the script.js file. Locate the following snippet of code:

We're first going to set up a chapter and go over how to properly name files. Locate the following snippet of code: 

//=// --- Chapter --- //=// [3]

let chapterArchive = [
    {
        name: '', // Archive Chapter name
        src: '', // HTML file prefix
        location: 'page', // Folder location relative to this file's location
        pages: null, // Insert the name of the array you made in the section before this

        chapter: '', // Comic page chapter name
        page: 'Page' // Page, episode, etc
    }
]

The array chapterArchive by default holds an empty chapter dictionary. Within that dictionary is the a list of key values that needs to be filled out for the archive and navigation system to properly work. Let's go through them!

The keys name and chapter are basically the same thing. They're both suppose to hold that chapter's name. The difference between the two is that name will be shown on the archive page, while chapter will be shown in all of the chapter pages' headers. It's recommended to keep chapter simple, while name can have some jazzy decorations.

The src is very important when it comes to naming files. All the files in that particular chapter NEED their names to begin with whatever the value for src.

The value for location is the relatvie folder location to script.js's location. By default, this value is "page". If were wanting to use a folder inside the page for your files, then the value would be "page/(folder name)".

The value for page is supposed to be whatever you wish your pages to be referred as. For example, you can have it be "Page" (Page 1), "Episode" (Episode 1), "Note" (Note 1), etc. You can be as creative or uncreative as you want. Like chapter, this will also appear in the page headers.

Hosting Your Site

Your site is just a simple static site, so it'll be easy to find free hosting plans for it. Here are some good platforms I'd suggest:

  • Netlify
    • Subdomain: *.netlify.app
    • Can use you custom domain for free.
    • Can host unlimited sites for free.
    • Storage: 1 GB
    • Bandwidth: 100 GB

  • Neocities
    • Subdomain: *.neocities.org
    • You have to be a monthly supporter to use your custom domain.
    • Can host one site for free.
    • Storage: 1 GB
    • Bandwidth: 100 GB
    If you choose Neocities, I'd find it nice if you'd add the tag deadjournals to your site.

  • GitHub Pages
    • Subdomain: *.github.io
    • Can use your custom domain for free.
    • Can host unlimited sites for free.
    • Storage: 1 GB
    • Bandwidth: 100 GB
